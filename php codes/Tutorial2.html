*php 가이드*


1. 상세보기 phphttp://csy9608.cafe24.com/Detail.php

  IN: boardID / OUT: boardID, boardDate, imagePath, imageTags


2. 로그인 php  http://csy9608.cafe24.com/Login.php

  IN: userID, userPassword / OUT: userID, success


3. 메인페이지 php (완성) http://csy9608.cafe24.com/MainPage.php

  IN: start, align_likes, temperature, limit / OUT: boardID, imagePath, imageTags, boardLikes, end

  웹으로 결과 한번 확인해보시면 뭔지 아실거에요,

  start에다 맨 처음에는 0을 넣으시고 그 다음에는 end로 받은 값을 넣어주시면 됩니다.

  align_likes에다 값을 넣으시면 좋아요순 정렬, 기본값은 최신순입니다. limit은 받아올 개수, temperature는 오늘의 평균온도


4. 마이페이지 php (완성) http://csy9608.cafe24.com/MyPage.php

  IN: userID, start, align_likes, limit / OUT: boardID, imagePath, imageTags, boardLikes, end

  메인페이지와 같은방식으로 동작합니다. 단지 userID가 생기고 temperature가 없는 차이


5. 회원가입 php  http://csy9608.cafe24.com/SignUp.php

  IN: userID, userPassword, userName, userAge, userGender / OUT: userID, success


6. 업로드 php (완성)  http://csy9608.cafe24.com/Upload.php

  IN: userID, temperature, encoded_string(이미지 base64 encoded string), imageTags / OUT: boardID, success

  boardID는 업로드된 게시물의 boardID


7. 아이디 확인 php  http://csy9608.cafe24.com/CheckId.php

  IN: userID / OUT: exists(아이디 존재여부)


8. 좋아요 php  http://csy9608.cafe24.com/UpdateLike.php

  IN: boardID, undo / OUT: success, boardLikes


9. 내 정보 수정 php  http://csy9608.cafe24.com/ModifyMyInfo.php

  IN: set, delete, userID, userName, (userPassword), userAge, userGender / OUT: (userID, userName, userAge, userGender), success

  수정은 set, 삭제는 delete 회원정보 보기가 디폴트, userPassword는 기존 비밀번호 확인 등을 한 뒤에 보낼 것, 안보내줘도 됨

  회원정보 보기인 경우에는 괄호 안의 값도 반환, 그 외에는(수정이나 삭제인 경우) 반환안함


10. 게시물 수정/삭제 php  http://csy9608.cafe24.com/ModifyBoard.php

  IN: userID, boardID, ( encoded_string, imageTags,) delete / OUT: success, authority

  기본적으로 수정모드, delete에 값을 보내면 삭제모드

  게시물 글쓴이가 현재 userID가 아닐 경우에 authority가 false, 접근 권한 없음. 수정 삭제 안됨

  현재 ID가 Admin인 경우 관리자이므로 authority 무조건 true
